TransE

前言

电子科技大学数据结构与算法课程任务

表示学习旨在学习一系列低维稠密向量来表示语言信息，而知识表示学习是面向知识库中实体和关系的表示学习。当今大规模知识库（或称为知识图谱）的构建为许多NLP任务提供了底层支持，但由于其规模庞大且不完备，如何高效存储和补全知识库成为了一项非常重要的任务，这就依托于知识表示学习。

TransE算法是经典的知识表示学习算法。

背景知识

一条知识图谱可以表示为一个三元组(sub,rel,obj)。前者是主体(h，头实体)，中间是关系(r)，后者是客体(t，尾实体)。主体和客体统称为实体（entity）。关系不可逆，也就是说主体和客体不能颠倒过来。知识图谱的集合，链接起来成为一个图（graph），每个节点是一个一个实体，每条边是一个关系，或者说是一个事实（fact）。也就是有向图，主体指向客体。

知识表示实际上是让知识图谱的实体和关系向量化。具体的说，就是将知识库中所有的实体、关系表示成一个低纬度的向量。即$(h,r,t)\rightarrow(\vec h,\vec r,\vec t)$。

TransE算法

TransE认为一个正确的三元组的编码满足$\vec h + \vec r=\vec t$，即头实体的编码加上关系的编码会等于尾实体的编码，而错误的三元组不满足这种关系。

于是定义了一个函数$d(\vec h+\vec r,~\vec t)$表示$\vec h+\vec r$与$\vec t$ 的距离，一般会选取向量的一范数或二范数。在设计损失函数时需要减小正确三元组的距离，增大错误三元组的距离，故损失函数可表示为下式： $$ loss=\sum_{(h,r,t\in S)}\sum_{(h',r,t'\in S')}[\gamma + d(h+r,t)-d(h'+r, t')]_+ $$ 其中

$S$ 表示正确的三元组集合

$S'$ 表示错误的三元组集合，通过随机替换正确三元组的头实体或尾实体，使之不存在于知识图谱中

$\gamma$表示正负样本之间的间距, 是一个常数

$[x]_+$ 表示$\max(0,x)$

通常为了方便训练避免过拟合，会加上约束条件，使实体与关系的编码向量长度小于等于1

而损失函数的梯度也易于求解，以二范数为例，损失函数对头实体的偏导为： $$ \frac{\partial loss}{\partial h_i}=\begin{cases}2(h_i+r-t) & \gamma+(h_i+r-t)^2-(h'+r-t')^2>0; \ 0 & \gamma+(h_i+r-t)^2-(h'+r-t')^2\leq 0\end{cases} $$

评分指标

该领域常用的评分指标有$mean~rank$和$hit@10$。

训练完后的实体entity，以及关系relation都是一维向量。对于某一测试数据$(h_0,r_0,t_0)$，将头实体或者尾实体替换成任意一种其它实体(n-1)个，这样得到$2*(n-1)$个新的三元组。接下来对这些三元组计算距离，再将他们进行升序排序，接下来就是计算两个评分指标了，对测试数据集中的数据都需要做如上操作。

对于$mean~rank$，在上述排好序的三元组中，正确的三元组的平均排名；

对于$hit@10$，在排好序的三元组中，如果前十个中有正确的三元组，则hit加一，最后计算hit的平均值除以测试数据总数，即正确三元组出现在前十的概率。

事实上，在上诉中构建的若干$n-1$的三元组中，可能有一些是存在于知识图谱中但不是测试的正确三元组，故上述方法低估了训练结果的能力，需要将这些三元组过滤，论文将不过滤得到的指标记为raw，将过滤后得到的指标记为flit，显然使用flit测试将会变得很慢。

计算评分指标代码如下

        for triple in self.test_triples:
            num += 1
			... # 中间省去对测试数据的过滤，更换头尾实体，计算距离等操作。
            rank_head_sorted = sorted(rank_head_dict.items(), key=operator.itemgetter(1), reverse=False)
            rank_tail_sorted = sorted(rank_tail_dict.items(), key=operator.itemgetter(1), reverse=False)
            # 计算hit_at_10和mean rank
            # 对于更换头实体的数据
            i = 0
            for i in range(len(rank_head_sorted)):
                if triple[0] == rank_head_sorted[i][0][0]:  # 找到真实数据
                    if i < 10:  # 在十以内
                        hits += 1
                    rank_sum = rank_sum + i + 1
                    break
            # 对于更换尾实体的数据
            i = 0
            for i in range(len(rank_tail_sorted)):
                if triple[2] == rank_tail_sorted[i][0][2]:  # 找到真实数据
                    if i < 10:   # 在十以内
                        hits += 1
                    rank_sum = rank_sum + i + 1
                    break
            if num % 5000 == 0:
                print(
                    "num: ",
                    num,
                    "hit_at_10:",
                    str(round(100 * hits / (2 * num), 3)) + '%',
                    "mean_rank:",
                    str(round(rank_sum / (2 * num), 3)),
                )
        self.hit_10 = hits / (2 * len(self.test_triples))
        self.mean_rank = rank_sum / (2 * len(self.test_triples))
        return self.hit_10, self.mean_rank
